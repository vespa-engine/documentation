---
title: "Zones"
---

An application is deployed to a *zone*, which is a combination of an [environment](/en/operations/environments) and a *region*, like `vespa deploy -z dev.aws-us-east-1c`.

If an application requires zone-specific configuration (e.g., different capacity requirements per zone), use [environment and region variants](/en/operations/deployment-variants#services.xml-variants). Also see [deployment.xml](/en/reference/applications/deployment).
