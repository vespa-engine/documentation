      <td>Visit</td>
      <td>Iterate over and get all documents, or a <a href="#selection" aria-label="selection">selection</a> of documents, in chunks, using <a href="#continuation" aria-label="continuation">continuation</a> tokens to track progress. Visits are a linear scan over the documents in the cluster. <code>{`/document/v1/`}</code> It is possible to specify namespace and document type with the visit path: <code>{`/document/v1/<namespace>/<document-type>/docid`}</code> Documents can be grouped to limit accesses to a subset. A group is defined by a numeric ID or string — see <a href="/en/schemas/documents#id-scheme" aria-label="id scheme">id scheme</a> . <code>{`/document/v1/<namespace>/<document-type>/group/<group>`}</code> <code>{`/document/v1/<namespace>/<document-type>/number/<number>`}</code> Mandatory parameters: <a href="#cluster" aria-label="cluster">cluster</a> - Visits can only retrieve data from &#42;one&#42; content cluster, so <code>{`cluster`}</code> &#42;&#42;must&#42;&#42; be specified for requests at the root <code>{`/document/v1/`}</code> level, or when there is ambiguity. This is required even if the application has only one content cluster. Optional parameters: <a href="#bucketspace" aria-label="bucketSpace">bucketSpace</a> - Parent documents are <a href="/en/reference/applications/services/content#document" aria-label="global">global</a> and in the <code>{`global`}</code> <a href="#bucketspace" aria-label="bucket space">bucket space</a>. By default, visit will visit non-global documents in the <code>{`default`}</code> bucket space, unless document type is indicated, and is a global document type. <a href="#concurrency" aria-label="concurrency">concurrency</a> - Use to configure backend parallelism for each visit HTTP request. <a href="#continuation" aria-label="continuation">continuation</a> <a href="#fieldset" aria-label="fieldSet">fieldSet</a> <a href="#selection" aria-label="selection">selection</a> <a href="#sliceid" aria-label="sliceId">sliceId</a> <a href="#slices" aria-label="slices">slices</a> - Split visiting of the document corpus across more than one HTTP request—thus allowing the concurrent use of more HTTP containers—use the <code>{`slices`}</code> and <code>{`sliceId`}</code> parameters. <a href="#stream" aria-label="stream">stream</a> - It's recommended enabling streamed HTTP responses, with the <a href="#stream" aria-label="stream">stream</a> parameter, as this reduces memory consumption and reduces HTTP overhead. <a href="#timeout" aria-label="timeout">timeout</a> <a href="#tracelevel" aria-label="tracelevel">tracelevel</a> <a href="#wanteddocumentcount" aria-label="wantedDocumentCount">wantedDocumentCount</a> <a href="#fromtimestamp" aria-label="fromTimestamp">fromTimestamp</a> <a href="#totimestamp" aria-label="toTimestamp">toTimestamp</a> <a href="#includeRemoves" aria-label="includeRemoves">includeRemoves</a> Optional request headers: <a href="#accept" aria-label="Accept">Accept</a> - specify the desired response format.</td>
