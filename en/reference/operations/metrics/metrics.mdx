## Metrics reference documentation

### Metric types

Each metric have one of the types documented [here](/en/reference/operations/metrics/metric-units).

### Metric aggregator suffixes

Metrics are collected over a time period so a metric reading must aggregate individual samples over this period. These are specified by adding the aggregator suffix to the metric name: <code>metric.name.aggregator</code>.

The following aggregators are available:

<table className="vespa-table">
  <thead>
    <tr>
      <th>Aggregator name (metric suffix)</th>
      <th>Explanation</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>95percentile</td>
      <td>The 95 percentile of samples in the period</td>
    </tr>
    <tr>
      <td>99percentile</td>
      <td>The 99 percentile of samples in the period</td>
    </tr>
    <tr>
      <td>average</td>
      <td>The average of samples in the period</td>
    </tr>
    <tr>
      <td>count</td>
      <td>The count of samples in the period</td>
    </tr>
    <tr>
      <td>last</td>
      <td>The last value sampled in the period</td>
    </tr>
    <tr>
      <td>max</td>
      <td>The max value sampled in the period</td>
    </tr>
    <tr>
      <td>min</td>
      <td>The min value sampled in the period</td>
    </tr>
    <tr>
      <td>rate</td>
      <td>The count of samples divided by the length of the period in seconds</td>
    </tr>
    <tr>
      <td>sum</td>
      <td>The sum of the sampled values in the period</td>
    </tr>
  </tbody>
</table>
