Welcome to Vespa! Following is a list of resources for first time users - Vespa can do many things, and be deployed in many ways. Let us know if something is missing.

Minimal Quick Start Install Vespa, insert a few documents and run a query, using Docker for or Vagrant:
  • Mac / Linux
  • Windows
  • Vagrant
For Vespa Cloud users, see Getting Started.
Monitoring Quick Start Try the Monitoring Quick Start - it builds on the Minimal Quick Start, and adds more features that most applications normally use:
  • Monitoring using a Grafana dashboard
  • High-performance feeding API
  • A document and query simulator - to validate the dashboard, but also something to build on for own application development
Plugin Development

Vespa supports writing plugins in languages that runs in the JVM. Use this to simplify Query and Document processing:

  • album-recommendation-java for Searcher development
  • album-recommendation-docproc for Document Processor development
Getting Started Vespa Applications has more details.

For Vespa Cloud users, also see Getting Started Custom Code.

Deployments

Vespa is an open source platform and can be deployed in multiple ways - explore:

  • quick-start-kubernetes (minikube) and basic-search-on-gke
  • quick-start-multinode-aws
  • quick-start-multinode-aws-ecs
These guides are good starters for building clusters of more than one node.

Also try deploying to Vespa Cloud, a managed service running Vespa.

Troubleshooting

Also see the FAQ.

No endpoint

Most problems with the quick start guides are due to Docker out of memory. Make sure at least 6G memory is allocated to Docker: 

$ docker info | grep "Total Memory"
OOM symptoms include 
INFO: Problem with Handshake localhost:8080 ssl=false: localhost:8080 failed to respond
The container is named vespa in the guides, for a shell do: 
$ docker exec -it vespa bash

Log viewing

Use vespa-logfmt to view the vespa log - example: 

$ /opt/vespa/bin/vespa-logfmt -l warning,error
