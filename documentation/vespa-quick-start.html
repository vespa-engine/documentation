  1. Clone vespa sample applications git repository on github.com: 

    $ git clone https://github.com/vespa-engine/sample-apps.git 
$ export VESPA_SAMPLE_APPS=/Users/foo/github/sample-apps     # replace with correct path

  2. Install Docker for Mac: Let Docker use 4GB in Preferences

  3. Start a Vespa Docker container: 

    $ docker run --detach --name vespa --hostname vespa-container --privileged \
  --volume $VESPA_SAMPLE_APPS:/vespa-sample-apps --publish 8080:8080 vespaengine/vespa

    The volume option makes the previously downloaded source code available, as /vespa-sample-apps, inside the docker container. We also publish port 8080 to make the search and feed interfaces accessible outside of the docker container.

  4. Log in to the Vespa Docker container:
    Using the vespa container name from the above docker run command, we can execute commands inside the container, e.g., start a bash shell with: 

    $ docker exec -it vespa bash

  5. Deploy and activate a sample application: 

    [root@vespa-container #/] vespa-deploy prepare /vespa-sample-apps/basic-search/src/main/application/ \
                          && vespa-deploy activate

    More sample applications can be found in sample-apps. Read more on applications in Application packages.

  6. Ensure the application is active:
    Wait for a valid 200 OK response from: 

    [root@vespa-container #/] curl --head http://localhost:8080/ApplicationStatus

  7. Feed documents: 

    [root@vespa-container #/] curl -s -X POST --data-binary @/vespa-sample-apps/basic-search/music-data-1.json \
                          http://localhost:8080/document/v1/music/music/docid/1 | python -m json.tool
[root@vespa-container #/] curl -s -X POST --data-binary @/vespa-sample-apps/basic-search/music-data-2.json \
                          http://localhost:8080/document/v1/music/music/docid/2 | python -m json.tool

    Read more on feeding and retrieving documents in the Document API.

  8. Run a search query and a single get request : 

    [root@vespa-container #/] curl -s http://localhost:8080/search/?yql=select%20%2A%20from%20sources%20%2A%20where%20default%20contains%20%22bad%22%3B | python -m json.tool 
[root@vespa-container #/] curl -s http://localhost:8080/document/v1/music/music/docid/2 | python -m json.tool

    Since we published port 8080 outside of the docker container, you can also view the results in a browser at http://localhost:8080/search/?yql=select%20%2A%20from%20sources%20%2A%20where%20default%20contains%20%22bad%22%3B. Read more in the Search API.

  • The Vespa APIs are useful to understand how to interface with Vespa.
